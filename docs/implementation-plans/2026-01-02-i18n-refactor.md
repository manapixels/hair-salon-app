# i18n Refactor and Legal Page Translations

**Date:** 2026-01-02

## Goal

Refactor the monolithic i18n JSON files into feature-specific namespaces to improve maintainability and performance. Implement bilingual Privacy Policy and Terms of Service pages to meet legal/platform requirements.

## Changes Implemented

### 1. i18n Structure Refactor

- **Old Structure**: Monolithic `src/i18n/en.json` and `src/i18n/zh.json`.
- **New Structure**:
  - `src/i18n/en/`: `common.json`, `dashboard.json`, `services.json`, `legal.json`
  - `src/i18n/zh/`: `common.json`, `dashboard.json`, `services.json`, `legal.json`
- **Configuration**: Updated `src/i18n/request.ts` to statically import and merge these files to maintain backward compatibility with existing usage (loading all keys into the message object) while enabling file separation.

### 2. Legal Pages

- **Extraction**: Hardcoded text from `src/app/[locale]/privacy/page.tsx` and `src/app/[locale]/terms/page.tsx` extracted to `legal.json`.
- **Translation**: Full Chinese translations added in `zh/legal.json`.
- **Implementation**: Updated pages to use `useTranslations('Legal.Privacy')` and `useTranslations('Legal.Terms')`.

## Verification

- Verified Development Server: `http://localhost:3001`
- Checked English and Chinese versions of Privacy and Terms pages.
- Verified Dashboard and Homepage still load correct translations (no regression).

## Next Steps

- Consider lazy loading namespaces in `request.ts` if bundle size becomes an issue, now that files are physically separated.
